## 缺陷详情记录

### BUG-001
- 日期：2026-05-09
- 模块：静态资源
- 严重级别：低
- 状态：已确认
- 发现人：OpenClaw / xiao
- 环境：本机 Linux Host，本地静态服务，Host Browser
- 关联测试项：Round 1 基础可运行性检查 / 首页加载无报错 / Round 4 局内功能级 smoke
- 复现步骤：
  1. 在项目目录启动本地静态服务。
  2. 打开 `http://127.0.0.1:18080/index.html`。
  3. 观察服务访问日志。
- 预期结果：页面相关静态资源请求全部正常返回。
- 实际结果：`/favicon.ico` 请求返回 404；Round 4 自动化 smoke 过程中浏览器 `console-error` 再次记录到同类 404。
- 影响范围：不影响游戏主流程、首页渲染、脚本加载与玩法逻辑。
- 临时绕过方案：可忽略；或后续补充 favicon 文件。
- 修复说明：待定。
- 修复版本：
- 复测结论：Round 4 自动化 smoke 复现同类 404，但首页加载、开始游戏、局内功能点击、重开本关、返回首页、限时模式进入均通过，确认该问题仍属低优先级静态资源问题。

### BUG-002
- 日期：2026-05-09
- 模块：测试环境
- 严重级别：低
- 状态：已确认
- 发现人：OpenClaw / xiao
- 环境：OpenClaw browser 工具 / Host Browser
- 关联测试项：Round 1 基础可运行性检查 / 自动化点击级 smoke test
- 复现步骤：
  1. 尝试通过 browser 工具执行 click / evaluate / console 检查。
  2. 调用 `act:click`、`act:evaluate` 或 console 相关能力。
  3. 观察工具返回结果。
- 预期结果：可直接自动执行浏览器交互与控制台检查。
- 实际结果：当前网关环境缺少完整 Playwright 支持，相关自动化交互能力不可用。
- 影响范围：不影响项目本身运行；影响自动化浏览器级 smoke test 与 console 检查效率。
- 临时绕过方案：先用静态检查 + 页面快照 + 手工交互补测。
- 修复说明：如需自动化浏览器交互，需补齐完整 Playwright 支持并重启网关。
